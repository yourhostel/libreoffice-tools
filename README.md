# libreoffice-tools

Набір макросів для LibreOffice, призначений для забезпечення контрольованої взаємодії 
з електронною таблицею та виключення прямого редагування даних. 
Рішення орієнтоване на автоматизацію роботи адміністратора хостелу
та реалізацію прикладних бізнес-процесів у середовищі LibreOffice Calc.

Проєкт реалізовано у вигляді макросів LibreOffice Basic із використанням форм введення даних. 
Форми побудовані на базових елементах інтерфейсу, таких як input, label, date, combo, button, 
забезпечуючи стандартизовану взаємодію користувача з системою.

Основна мета системи полягає у перенесенні логіки роботи з даними з рівня таблиці
на рівень керованого інтерфейсу, для цілісності, контролю доступу та аудиту змін.

## Функціональні можливості

- Система забезпечує ведення обліку заселення шляхом створення записів у таблиці через форму введення.

- Реалізовано механізми пошуку записів за ключовими атрибутами:
    - ім'ям та по-батькові гостя
    - номером телефону
    - номером ліжко-місця
    - кімнатою
    - ідентифікацією адміністратора, який виконав операцію
    - статусом наявності у чорному списку

- Передбачено автоматичне заповнення персональних даних на основі ідентифікатора запису.

- Реалізовано контроль перетину дат проживання, що виключає можливість створення конфліктних записів для одного місця у заданому часовому інтервалі.

- Система виконує моніторинг строків проживання та автоматично виділяє записи, для яких наближається завершення періоду перебування.

- Передбачено механізм додавання клієнтів до чорного списку з одночасним звільненням займаного місця.

- Реалізовано обробку виключних ситуацій при введенні некоректних даних із відображенням повідомлень користувачу.

- Система забезпечує авторизований доступ трьох рівнів:

    - користувач:
        - заборона прямого редагування таблиці, зміна даних дозволена виключно через форми
        - заборона редагування після закриття звітного періоду
        - обмежена область відображення історії заселень, що визначається конфігурацією

    - привілейований користувач:
        - дозволено змінювати записи у довільному діапазоні дат
        - можливість призначення користувачів на зміну
        - формування звітів
        - повна область відображення даних

    - super:
        - доступ через зміну конфігурації
        - повний доступ на рівні LibreOffice Calc

- Передбачено ведення журналу подій, у якому фіксуються операції видалення записів,
  це забезпечує контроль несанкціонованих змін.

## Приклади реалізованих форм

Форми побудовані з компонентів, що можна використовувати повторно. Це дозволяє збирати нові форми
без дублювання коду й змінювати ті, що існують без побічних ефектів.
Компоненти можна комбінувати під різні сценарії роботи адміністратора. Додавання нової форми зводиться до
композиції цих компонентів.

## Встановлення

Скрипт deploy.sh виконує розгортання макросів у середовище LibreOffice.
Він розпаковує контейнер із модулями та копіює їх у стандартний каталог користувача:

```
$HOME/.config/libreoffice/4/user/basic/Standard
```

Це дозволяє перетворити текстові файли формату .bas у внутрішній формат LibreOffice .xba.

Для виконання необхідно задати змінну ODS, яка вказує на файл-контейнер модулів, після чого виконати:

```bash
chmod +x deploy.sh
./deploy.sh
```

## Особливості формату макросів

Формат .xba є XML-представленням коду, що вимагає екранування спеціальних символів.
При конвертації з .bas необхідно враховувати такі відповідності:

| Символ | Назва          | Екранована форма |
|--------|----------------|------------------|
| `'`    | апостроф       | `&apos;`         |
| `"`    | подвійна лапка | `&quot;`         |
| `&`    | амперсанд      | `&amp;`          |
| `<`    | менше          | `&lt;`           |
| `>`    | більше         | `&gt;`           |

### Приклад:

- `.bas`

```
' =====================================================
' === Функція IsCancelCode =============================
' =====================================================
Function IsCancelCode(nCode As Long) As Boolean
    IsCancelCode = InStr(" 20 21 22 23 ", " " & nCode & " ") > 0
End Function
```

- `.xba`

```
&apos; =====================================================
&apos; === Функція IsCancelCode =============================
&apos; =====================================================
Function IsCancelCode(nCode As Long) As Boolean
    IsCancelCode = InStr(&quot; 20 21 22 23 &quot;, &quot; &quot; &amp; nCode &amp; &quot; &quot;) &gt; 0
End Function
```

## Python провайдер

> *( Заплановано як тестовий проєкт. Реалізація на Python зараз відсутня )*

Для використання макросів на Python необхідно встановити відповідний провайдер:

```bash
sudo snap remove libreoffice
sudo apt remove --purge libreoffice*
sudo add-apt-repository ppa:libreoffice/ppa
sudo apt update
sudo apt install libreoffice libreoffice-script-provider-python
```

Перевірка встановлення:

```bash
dpkg -l | grep libreoffice-script-provider-python
```

## Призначення

Система розроблена на основі вимог реального бізнес-процесу управління хостелом
і використовується як приклад прикладної реалізації у курсовому проєкті.
Основна цінність полягає у демонстрації підходу до побудови контрольованих інтерфейсів доступу
до табличних даних без використання повноцінної серверної системи.
